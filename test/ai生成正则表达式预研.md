# 正则表达式智能生成预研

## 一、相关应用或产品

### 1.1 AutoRegex
- **网址**：https://www.autoregex.xyz/
- **特点**：可以通过自然语言生成正则表达式
- **局限性**：
  - 对中文不友好，主要支持英文输入
  - 为开源项目，但功能相对简单
  - 无法处理复杂的多场景需求

### 1.2 Octopus
- **特点**：国内工具，由机器学习训练
- **局限性**：
  - 效果不佳，不如使用LLM模型
  - 近期不再维护，缺乏持续更新
  - 未开源，无法进行二次开发
  - 无法处理复杂的多语言场景

### 2.2 核心算法

系统采用迭代优化算法，通过多轮交互逐步优化正则表达式：

1. **初始生成**：根据用户需求生成初始正则表达式
2. **验证与反馈**：使用Python的re模块验证生成的正则表达式
3. **错误处理**：捕获语法错误和执行超时等异常
4. **迭代优化**：将验证结果反馈给LLM进行修正
5. **最终确认**：在满足条件时返回最终结果

### 4.3 错误分析

**语法错误**（占比30%）：
- `unknown extension ?\d`：LLM生成的语法不符合Python re模块
- `multiple repeat`：重复修饰符使用错误
- `unbalanced parenthesis`：括号不匹配

**语义错误**（占比50%）：
- 匹配范围过宽：匹配了不应匹配的内容
- 匹配范围过窄：遗漏了应匹配的内容
- 边界处理不当：在长字符串中部分匹配

**性能错误**（占比20%）：
- 执行超时：复杂正则表达式导致性能问题
- 内存溢出：极复杂模式消耗过多资源

## 五、接口使用

API接口地址：`http://xx.xx.xx.xx:8765/generate-regex`

### 参数说明

- [user_request] (string, required): 用户对正则表达式的自然语言描述，明确说明需要匹配的内容类型
- [sample_input] (string, required): 包含目标匹配内容的示例文本，用于模型理解边界信息
- [expected_matches] (array, required): 期望从user_request示例文本中匹配到的字符串数组，作为正确匹配的标准
- [negative_examples] (array, optional): 不应该被匹配的示例文本数组，用于防止误匹配（非必须填）
- [additional_examples] (array, optional): 额外的示例文本数组，提供更多匹配模式的参考（非必须填）

### 请求示例
```json
{
    "user_request": "获取号码",
    "sample_input": "Call me at (555) 123-4567 to schedule an appointment",
    "expected_matches": [
      "(555) 123-4567"
    ],
    "negative_examples": [
      "555",
      "1234567890123456",
      "phone number"
    ],
    "additional_examples": [
      "用户的电话是 (785) 139-5563"
    ]
  }
```

### 返回示例
```json
{
    "success": true,
    "regex": "\\(\\d{3}\\) \\d{3}-\\d{4}",
    "matches": [
        "(555) 123-4567"
    ],
    "attempts": 2,
    "error": null
}
```
